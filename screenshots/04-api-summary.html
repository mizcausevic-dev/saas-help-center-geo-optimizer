api summary

An API layer that makes content optimization measurable and easy to integrate.

The service exposes summaries, article-level scores, and prioritized fixes so RevOps, support, and content teams can use the same GEO queue in dashboards, internal tools, or publishing workflows.

routes

Summary, queue, and article scoring APIs.

/api/dashboard/summary, /api/articles, /api/articles/{slug}, and /api/sample.

fit

Built for SaaS content ops and AI-era support.

It works well as a content QA layer for help centers, onboarding docs, and release-note libraries.
